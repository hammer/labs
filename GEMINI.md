# Gemini CLI - Project Guidance

## Project Overview
This project is an **AI Lab Tracker** that monitors China's leading AI research labs, their flagship models, and research outputs. It is built as a static site using **Astro**, **TypeScript**, and **Tailwind CSS** (though some components might use Vanilla CSS).

## Core Data Structure
The project is data-driven, with all information stored in YAML files:
- **Labs:** `data/labs/*.yaml`. Schema defined in `src/schema.ts` (LabSchema).
- **Outputs:** `data/outputs/**/*.yaml`. Schema defined in `src/schema.ts` (OutputSchema).
- **Metrics:** `data/metrics.json`. Auto-generated by `npm run fetch-metrics`.

### Key Schema Rules:
- **Slugs:** Must be lowercase alphanumeric with hyphens (e.g., `deepseek-v3`).
- **Dates:** Must follow `YYYY-MM-DD` or `YYYY-MM` formats as specified in the schema.
- **Output Types:** `model`, `paper`, `blog`, `library`, `dataset`, `announcement`.
- **References:** Outputs must reference valid lab slugs and can reference other output slugs (e.g., `base_model`, `related`).

## Development Workflow
Always run these commands to ensure data integrity and site functionality:
- `npm run dev`: Start the development server (localhost:4321).
- `npm run validate`: **CRITICAL.** Always run this after modifying YAML data.
- `npm run fetch-metrics`: Update impact metrics (stars, downloads, citations).
- `npm run build`: Build the static site for production.

## Canonical Identifiers (DESIGN_NOTES.md)
When adding new models, papers, or libraries, prioritize collecting:
- **Hugging Face:** Model/Dataset IDs.
- **arXiv:** Paper IDs (e.g., `2412.19437`).
- **GitHub:** `owner/repo`.
- **Artificial Analysis:** Provider/model slugs.

## Logo Management
Logos should be stored in `public/logos/` as PNG files, named after the lab slug (e.g., `alibaba.png`).

## Code Standards
- Use **TypeScript** for all scripts and components.
- Adhere to **Zod** schemas for data validation.
- Maintain consistency with existing component patterns in `src/components/`.
